Architecture Overview and Data Flow Chart.

Goal-oriented dialogue systems is a category of dialogue systems designed to solve one or multiple specific goals or tasks (e.g. flight reservation, hotel reservation, food ordering, appointment scheduling). Traditionally, goal-oriented dialogue systems are set up as a pipeline with four main modules: 1-Natural Language Understanding (NLU), 2-Dialogue State Tracking (DST), 3-Dialog Policy Manager, and 4-Response Generator. NLU extracts the semantic information from each dialogue turn which includes e.g. user intents and slot values mentioned by the user or system. DST takes the extracted entities to build the state of the user goal by aggregating and tracking the information across all turns of the dialogue. Dialog Policy Manager is responsible for deciding the next action of the system based on the current state. Finally, Response Generator converts the system action into human natural text understandable by the user.




This is CAiFE high level architecture.

In the offline mode, models will be trained on available datasets and kept ready for further use. Using the Caife portal, a business owner can initiate meta learning for the domain the business would like to use. Data can be supplied and model will train using that data.

Business owner will also have to supply knowledge base data in pre-defined format so it can be used to make the chatbot business specific. If no training data is available, a model based on selected domain will be chosen. Model and knowledge base will be integrated to create a URL that can be used in website.

Technology Overview

Here is a Natural language understanding (NLU) module. In this project, we use TRADE-DST, which stands for transferable dialogue state generator. This model is essentially a LSTM encoder model which converts user text into intents and slots, this works even for those unseen texts. The dialog state tracker keeps a record of slots history and current state.With a dialog management (DM) system, developers can control how the bot navigates between tasks, i.e., how it behaves when required to pause a particular task, switch intent, complete a different task, and then return to the original user request without losing context.

The next module is Natural Language Generation (NLG) module converts a dialog act represented in a semantic form into a response in natural language. We use GPT-2 model, which is Generative Pre-trained Transformer 2 (GPT-2) is an open-source artificial intelligence created by OpenAI in February 2019. It is basically a machine learning model that is able to look at part of a sentence and predict the next word. Since it is a pre-trained model, we just did a fine-tuning here with 50 pairs of dialogues.
